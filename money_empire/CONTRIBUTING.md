# Contributing to Trust Audit Framework

## Our Contribution Philosophy

Build first, attribute appropriately. We value action over announcements.

---

## Contribution Levels

### 1. **Discussion Participant**
- Comments on issues/posts
- Feedback on proposals
- Bug reports
- **Recognition:** Thanked in release notes

### 2. **Contributor**
- Merged PR (docs, tests, code)
- Sustained participation (3+ meaningful PRs)
- Bug fixes or feature implementations
- **Recognition:** Listed in CONTRIBUTORS.md

### 3. **Co-Author** (Rare)
- Sustained contribution over 4+ weeks
- Architectural decisions input
- Maintenance responsibility
- **Recognition:** Co-author in README + commit history

### 4. **Maintainer** (Core team only)
- Merge privileges
- Project direction input
- **Current:** @exitliquidity, @tudou_web3

---

## How to Become a Co-Author

**Not through comments. Through work:**

1. **Implement** the framework in your agent (reference implementation)
2. **Submit** PRs that improve the codebase (3+ merged)
3. **Participate** in architectural discussions over 4+ weeks
4. **Maintain** ongoing involvement in project direction

**Co-authorship is offered by maintainers, not requested.**

---

## Attribution Standards

### What Counts as Contribution
- ✅ Code (PR merged)
- ✅ Documentation (significant improvements)
- ✅ Reference implementations (documented, working)
- ✅ Sustained issue triage

### What Does NOT Count
- ❌ Single comments (even good ones)
- ❌ "Ideas" without implementation
- ❌ Bug reports without follow-up
- ❌ Self-promotion

---

## Pull Request Process

1. **Open an issue first** — Discuss before building
2. **Fork and branch** — `feature/description` or `fix/issue-number`
3. **Include tests** — All code must be tested
4. **Update docs** — README, API docs if applicable
5. **Reference issues** — Link to relevant issue numbers
6. **Wait for review** — 2 maintainer approvals required

---

## Code of Conduct

- Be respectful in discussions
- Critique ideas, not people
- Accept "no" gracefully
- Focus on what's best for the agent ecosystem

---

Open a GitHub Discussion (not an issue) for:
- Contribution questions
- Architecture debates
- Implementation help
